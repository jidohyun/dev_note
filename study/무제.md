## SQL 관련 키워드

- CREATE: 새 테이블이나 데이터베이스를 생성할 때 쓰는 명령어. 예) CREATE TABLE.[](https://dev.mysql.com/doc/en/keywords.html)​
- CREATE_AT: 생성된 시각을 저장하는 컬럼 이름(보통 created_at 같은 용도).[](https://gengminy.tistory.com/8)​
    
- CURRENT_DATE: 오늘 날짜를 기본값으로 넣는 함수/키워드.[](https://stackoverflow.com/questions/20461030/current-date-curdate-not-working-as-default-date-value)​
    
- DATE: 날짜만 저장하는 자료형.[](https://dev.mysql.com/doc/en/keywords.html)​
    
- DEFAULT: 값이 안 들어왔을 때 자동으로 채울 기본값을 지정하는 제약 조건.[](https://www.w3schools.com/sql/sql_ref_default.asp)​
    
- DESC: ORDER BY와 같이 써서 내림차순(큰 값 → 작은 값) 정렬.[](https://dev.mysql.com/doc/en/keywords.html)​
    
- FROM: 어떤 테이블에서 데이터를 가져올지 지정하는 키워드.[](https://dev.mysql.com/doc/en/keywords.html)​
    
- INT: 정수형 숫자 자료형(간단한 정수).[](https://dev.mysql.com/doc/en/keywords.html)​
    
- INTEGER: 정수형 숫자 자료형(표기만 다르고 INT와 비슷하게 사용).[](https://dev.mysql.com/doc/en/keywords.html)​
    
- INTO: INSERT 문에서 “어느 테이블에 넣을지” 지정할 때 사용.[](https://www.w3schools.com/sql/sql_ref_default.asp)​
    
- NOT: NOT NULL 같은 식으로 “아니다/허용 안 함” 뜻으로 사용.[](https://www.w3schools.com/sql/sql_ref_default.asp)​
    
- NUMBER: 숫자 전체(정수/실수)를 표현하는 자료형 이름(오라클 등에서 사용).[](https://dev.mysql.com/doc/en/keywords.html)​
    
- PRIMARY KEY: 테이블에서 각 행을 유일하게 식별하는 기본 키 제약 조건.[](https://www.w3schools.com/sql/sql_ref_default.asp)​
    
- REFERENCES: 외래 키가 어느 테이블·컬럼을 참조하는지 지정.[](https://junu128.tistory.com/74)​
    
- TABLE: 테이블을 만들거나 가리킬 때 쓰는 키워드.[](https://dev.mysql.com/doc/en/keywords.html)​
    
- VALUES: INSERT에서 실제로 넣을 값들을 나열하는 부분.[](https://www.w3schools.com/sql/sql_ref_default.asp)​
    
- VARCHAR: 가변 길이 문자열 자료형.[](https://dev.mysql.com/doc/en/keywords.html)​
    

## JavaScript / 일반 키워드

- allowNull: 컬럼에 NULL 허용 여부를 설정하는 옵션(false면 NOT NULL).[](https://gengminy.tistory.com/8)​
    
- async: 비동기 함수를 선언할 때 쓰는 키워드(함수는 Promise를 반환).[](https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Statements/async_function)​
    
- attributes: 모델이 가지고 있는 컬럼(필드)들을 나타내는 속성.[](https://gengminy.tistory.com/8)​
    
- autoIncrement: 숫자 컬럼을 자동으로 1씩 증가시키는 옵션.[](https://gengminy.tistory.com/8)​
    
- console: console.log 등 로그를 출력하는 JS 내장 객체.[](https://www.w3schools.com/js/js_reserved.asp)​
    
- const: 재할당할 수 없는 변수(상수)에 사용하는 키워드.[](https://www.w3schools.com/js/js_reserved.asp)​
    
- db: 데이터베이스 연결 객체를 의미하는 변수 이름으로 자주 사용.[](https://gengminy.tistory.com/8)​
    
- defaultValue: 값이 없을 때 자동으로 채울 기본값을 지정하는 Sequelize 옵션.[](https://gengminy.tistory.com/8)​
    
- init: Sequelize에서 모델 클래스를 초기화(컬럼 정의+옵션 설정)하는 정적 메서드.[](https://gengminy.tistory.com/8)​
    
- map: 배열의 각 요소를 변환해서 새 배열을 만드는 JS 배열 메서드.[](https://www.w3schools.com/js/js_reserved.asp)​
    
- order: 정렬 기준을 나타내는 옵션(ORDER BY나 Sequelize order 옵션 등).[](https://youthangcode.tistory.com/entry/Nodejs-%EA%B5%90%EA%B3%BC%EC%84%9C-7%EC%9E%A5-MySQL-%EC%A0%95%EB%A6%AC%ED%95%98%EA%B8%B0)​
    
- static: 클래스의 정적 메서드/속성을 정의하는 키워드(인스턴스가 아니라 클래스에서 직접 호출).[](https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Classes/static)​
    
- super: 부모 클래스(상속한 상위 클래스)의 생성자나 메서드를 호출할 때 사용.[](https://stackoverflow.com/questions/26255/reserved-keywords-in-javascript)​
    
- throw: 예외를 던질 때 사용하는 키워드.[](https://www.w3schools.com/js/js_reserved.asp)​
    
- type: 컬럼의 자료형(STRING, INTEGER 등)을 지정하는 속성 이름.[](https://gengminy.tistory.com/8)​
    

## Sequelize 모델 옵션/관계 키워드

- belongsTo: 한 모델이 다른 모델에 속해 있다는 관계(외래 키가 자신 쪽에 있음).[](https://thelapssql.tistory.com/62)​
    
- foreignKey: 관계에서 “내가 가지고 있는 외래 키 컬럼 이름”을 지정.[](https://junu128.tistory.com/74)​
    
- targetKey: 관계에서 “상대 모델의 어떤 컬럼을 참조할지” 지정(기본은 id).[](https://junu128.tistory.com/74)​
    
- modelName: Sequelize에서 모델(클래스)의 이름을 지정하는 옵션.[](https://sskey.tistory.com/69)​
    
- order: findAll 등에서 정렬 순서를 지정하는 옵션 배열.[](https://youthangcode.tistory.com/entry/Nodejs-%EA%B5%90%EA%B3%BC%EC%84%9C-7%EC%9E%A5-MySQL-%EC%A0%95%EB%A6%AC%ED%95%98%EA%B8%B0)​
    
- paranoid: true면 삭제 시 실제 삭제 대신 deletedAt에 시간만 기록(소프트 삭제).[](https://loy124.tistory.com/294)​
    
- primaryKey: 해당 컬럼이 기본 키인지 여부를 나타내는 옵션.[](https://gengminy.tistory.com/8)​
    
- require: Node.js에서 모듈을 불러올 때 사용하는 함수(require('모듈')).[](https://gengminy.tistory.com/8)​
    
- tableName: 실제 DB 테이블 이름을 명시적으로 지정하는 옵션.[](https://sskey.tistory.com/69)​
    
- underscored: true면 created_at 같이 스네이크 케이스 컬럼명 사용, false면 createdAt 같은 카멜 케이스.[](https://youthangcode.tistory.com/entry/Nodejs-%EA%B5%90%EA%B3%BC%EC%84%9C-7%EC%9E%A5-MySQL-%EC%A0%95%EB%A6%AC%ED%95%98%EA%B8%B0)​
    
- utf8: 데이터베이스/테이블 문자 인코딩을 UTF-8로 사용하겠다는 설정.[](https://gengminy.tistory.com/8)​
    
- utf8_general_ci: UTF-8용 일반 정렬 규칙(collation) 설정 값.[](https://sskey.tistory.com/69)​
    
- where: SELECT/UPDATE/DELETE에서 조건(필터)을 지정하는 부분 또는 옵션.
