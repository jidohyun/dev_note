### 개요

브라우저는 HTML·CSS·JS를 해석해 DOM과 CSSOM을 만든 뒤 이를 결합해 Render Tree를 구성하고, 레이아웃과 페인팅 과정을 거쳐 화면에 그린다.

### 렌더링 순서

1. **HTML 파싱 → DOM 생성**
2. **CSS 파싱 → CSSOM 생성**
3. **DOM + CSSOM → Render Tree 생성**
4. **레이아웃(Reflow)**: 각 요소의 위치·크기 계산
5. **페인팅(Paint)**: 색상·테두리 등 실제 픽셀로 렌더링

### script 태그의 영향

- HTML 파싱 중 script를 만나면 브라우저는 **파싱을 멈추고 스크립트를 로드·실행**한다.
- DOM 생성이 지연되고 성능 문제가 생길 수 있어 async·defer 속성으로 이를 완화한다.

### async vs defer

- **공통**: 스크립트를 비동기로 가져와 파싱 중단을 줄인다.
- **async**: 로드되는 순간 **즉시 실행**, 순서 보장 없음 → 독립 스크립트용.
- **defer**: HTML 파싱이 끝난 직후 **문서 순서대로 실행** → 의존성이 있는 스크립트용.

### 면접 포인트(짧게)

“브라우저는 DOM·CSSOM을 만든 뒤 Render Tree를 생성해 레이아웃과 페인팅을 수행한다. script는 파싱을 중단시키므로 async/defer로 영향 최소화한다. async는 즉시 실행, defer는 파싱 완료 후 순서대로 실행된다.”
