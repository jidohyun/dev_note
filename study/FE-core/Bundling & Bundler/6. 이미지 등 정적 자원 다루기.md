이미지, 아이콘, 폰트같은 파일들은 브라우저에서 JS나 CSS코드로 변환되지 않고 그대로 사용되는 정적 파일들이다.

웹팩의 `Asset Modules`를 활용하면 이런 정적 자원들도 JS 모듈처럼 import해서 쓸 수 있음.

---
### Asset Modules?

이전엔 이미지나 폰트같은 정적 자원들을 웹팩에서 처리하기 위해 `file-loader`, `url-loader`, `raw-loader`같은 별도 로더를 설치함.

웹팩 5부터는 이러한 작업들을 쉽게 해주는 `Asset Modules`가 내장됌. 이젠 따로 로더를 설치하지 않아도 웹팩으로 이미지, 폰트, 아이콘 같은 파일을 자동으로 처리하고, 필요하면 압축하거나 base 64 형태로 변환해서 JS코드 안에 바로 포함시켜주기도 함.

---
### 웹팩에 Asset Modules 설정 추가하기

```js
module.exports = {
  // ... 기존 설정 유지
  module: {
    rules: [
      // ... 기존 rules 유지
      {
        test: /\.(png|svg|jpg|jpeg|gif)$/i, // 이미지 파일 확장자
        type: "asset" // Asset Modules 사용
      }
    ]
  }
};
```

### 이미지 파일 JS에서 import하기

`App.tsx`에서 logo 파일을 import하고 로고 이미지 태그 `src`에 넣음.

```tsx
// logo.svg 를 import로 가져옴
import logo from "./assets/logo.svg";

// 가져온 logo 모듈을 src에 넣어줌
<img src={logo} alt="Logo" className="logo"></img>;
```
