
번들러는 여러 파일을 하나로 묶는 번들링을 수행할 뿐만 아니라, 코드를 브라우저가 이해하기 쉽게 변환하고 최적화해서 웹사이트가 더 빨리 로딩되도록 도와주는 도구임.

---

### 번들러가 하는 일

#### 1. 파일 묶기 (Bundling)

- 여러 개의 파일을 하나(또는 몇 개)로 묶어 네트워크 요청 수를 줄여 페이지 로딩 속도를 빠르게 만들어줌.
- 모듈 시스템(ES Modules, CommonJS 등)을 자유롭게 사용 가능해짐.

#### 2. 코드 변환 (Transpiling)

- TS, JSX 등의 코드를 브라우저가 이해할 수 있는 JS로 변환함.
- 브라우저 지원을 걱정하지 않고 최신 언어 기능을 쓸 수 있음. (Babel 같은 도구를 같이 씀)

#### 3. 최적화 (Optimization)

- 사용하지 않는 코드를 제거함. ([[Tree shaking]])
- 공백, 주석 제거 등 코드를 압축함. ([[Minification]])
- 초기 로딩을 빠르게 하기 위해 필요한콛 만 별도로 나눔. ([[Code splitting]])
- 필요할 때 파일을 동적으로 불러옴. ([[Lazy loading]])
- 이미지를 자동으로 최적화해서 용량을 줄여줌.

#### 4. 개발 환경 제공

- 개발할 때 파일을 수정하면 페이지를 새로고침하지 않고도 수정사항을 즉시 반영해줌. (Hot Module Replacement)
- 개발자가 디버깅할 때 원본 코드와 번들된 코드를 쉽게 비교할 수 있도록 소스맵을 만들어줌.
