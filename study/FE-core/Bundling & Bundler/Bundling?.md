번들링은 여러 개의 파일(JS, CSS, 이미 등 웹 개발에 필요한 리소스)을 몇 개의 파일로 묶는 작업임. 이렇게 묶인 파일을 번들(Bundle)이라고 한다.

![](https://i.imgur.com/HbdoLfF.png)

웹 애플리케이션을 만들다 보면 JS or TS 파일이 수백개로 쪼개진다. 각 파일은 기능 단위로 잘게 나눠지기 때문에 개발하기는 편하지만, 브라우저가 서버로부터 파일의 수만큼 요청을 보내게 되어 네트워크 비용이 커지고 로딩 속도가 느려짐.

번들링은 위의 문제를 해결하기 위한 방법.

---

### 번들링의 목적

1. 요청 수 감소: 수백개의 파일을 하나로 묶어 브라우저가 요청해야 할 파일의 수를 줄여서 로딩 속도를 향상시킴.
2. 캐싱 최적화: 묶인 파일 하나만 캐시하면 되서 효율적임.
3. 유지보수성과 배포 효율성: 개발할 때는 모듈화를 유지하며, 배포할 땐 성능 최적화를 할 수 있음.

---

### 번들링 과정 한눈에 보기

#### 1. 여러 개의 JavaScript 파일

```
├── index.js
├── utils.js
├── auth.js
└── dashboard.js
```

#### 2. 파일들이 서로 의존

`index.js`가 `auth.js`와 `dashboard.js`를 불러오고, `auth.js`는 `utils.js`를 사용하고 있을 수 있음. 즉, 파일 간에 의존성 관계가 생김.

#### 3. 번들러가 파일 관계를 분석

번들러는 프로젝트 안의 파일들을 스캔하며 누가 누구를 쓰는지 분석함. 시작 지점(예: `index.js`)부터 출발해서 모든 필요한 파일을 찾고, 의존성 그래프를 그림.

#### 4. 하나의 파일로 묶음 (Bundling)

필요한 파일들을 의존성 순서에 맞춰 하나의 파일로 합침.

```
└── bundle.js
```

#### 5. 번들러의 추가 최적화 작업

- 사용되지 않는 코드는 제거. ([[트리 세이킹]])
- 필요한 경우, 여러 개의 작은 번들로 나눔. ([[코드 스플리팅]])
- 코드에서 공백, 주석을 없애서 크기를 줄임.

#### 6. 최종 결과물을 배포

최적화된 `bundle.js` 파일을 서버에 올리고, 사용자는 브라우저로 빠르게 접근할 수 있음

---

### 결론

현대 번들링 도구들은 단순히 여러 파일을 하나로 합치는 것에 그치지 않고, 코드를 더 빠르게 동작하도록 압축하던가, 사용하지 않는 코드를 제거하는 등 다양한 최적화도 자동으로 해줌
