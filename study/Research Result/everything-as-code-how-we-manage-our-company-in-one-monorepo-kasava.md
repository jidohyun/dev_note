# 핵심 요약
- 모든 회사 자산(코드, 마케팅, 문서, 투자 자료 등)을 하나의 모노레포에서 관리하여 운영 효율을 극대화함.
- AI에게 전체 맥락(Context)을 제공함으로써 AI가 코드 변경에 맞춰 문서와 마케팅 자료를 동시에 수정하도록 하는 'AI 네이티브 개발' 실천.
- 단일 커밋과 'git push'만으로 서비스 전반(백엔드, 프론트엔드, 웹사이트, 문서)의 일관성을 유지하고 즉시 배포하는 문화 구축.
- 복잡한 도구(CMS, 별도 문서 툴) 대신 코드 중심의 워크플로우를 통해 팀 전체의 실행 속도를 높임.

# 상세 내용
- **AI 네이티브 개발 (AI-Native Development)**
  - AI는 맥락이 중요하며, 모노레포는 회사 전체의 정보를 AI에게 한 번에 제공할 수 있는 최적의 구조임.
  - AI가 코드를 읽고 문서를 작성하거나, 백엔드 로직을 바탕으로 마케팅 문구의 사실 여부를 검증할 수 있음.
  - "가격 정책 변경"과 같은 요청 시 AI가 백엔드, 프론트엔드, 마케팅 페이지, 문서를 한 번에 업데이트함.

- **원자적 변경 (Atomic Changes)**
  - 백엔드 API 변경 시 프론트엔드 타입 정의와 관련 문서를 동일한 커밋에서 수정하여 버전 불일치 문제를 원천 차단.
  - 단일 소스(예: billing-plans.json)를 참조하여 앱과 웹사이트의 정보가 항상 일치하도록 설계.

- **모든 것의 코드화 (Everything as Code)**
  - 마케팅: 블로그 포스트(Markdown), 이메일 템플릿(MJML), 투자자 덱(Next.js)을 코드로 관리.
  - 문서: 공개 문서(Mintlify)와 내부 아키텍처 문서를 코드와 나란히 배치하여 검색 및 최신화 용이성 확보.
  - 외부 서비스: 크롬 확장 프로그램, 구글 독스 부가기능 등도 동일 레포에서 관리하여 API 계약(Contract) 공유.

- **기술적 구현 및 운영**
  - 단순성 유지: 복잡한 npm/yarn 워크스페이스 대신 각 디렉토리를 독립적인 npm 프로젝트로 관리하여 의존성 혼란 방지.
  - 선택적 CI/CD: GitHub Actions의 `paths` 필터를 사용하여 변경된 부분만 테스트하고 배포함으로써 효율성 증대.
  - AI 가이드: 각 주요 디렉토리에 `CLAUDE.md`를 두어 AI가 해당 프로젝트의 기술 스택과 규칙을 즉시 파악하게 함.

# 인사이트 / 할 일
- **인사이트**: AI 시대의 생산성은 단순히 코드를 빨리 짜는 것이 아니라, AI가 비즈니스 전체의 맥락을 얼마나 쉽고 정확하게 파악할 수 있는 구조를 만드느냐에 달려 있음.
- **인사이트**: 'git push'를 유일한 배포 수단으로 삼는 것은 비엔지니어링 직군에게는 진입장벽일 수 있으나, 장기적으로는 관리 포인트와 동기화 오류를 획기적으로 줄여줌.
- **할 일**: 현재 파편화된 문서(Notion), 마케팅 자료, 코드를 연결할 수 있는 방안 검토.
- **할 일**: 프로젝트 레포지토리에 AI를 위한 가이드 파일(예: CLAUDE.md 또는 AI.md)을 작성하여 협업 효율 테스트하기.
