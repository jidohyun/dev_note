# 핵심 요약
- 앤스로픽(Anthropic)이 출시한 'Claude Code'는 복잡한 마법이 아니라, 기존 LLM의 도구 사용(Tool Use) 기능을 활용한 CLI 래퍼(Wrapper)에 불과함.
- 약 200줄의 파이썬 코드로 Claude Code의 핵심 기능인 파일 읽기, 쓰기, 검색, 터미널 실행 등을 동일하게 구현할 수 있음.
- 에이전트의 핵심은 '추론-행동(ReAct)' 루프이며, 이는 모델이 도구를 선택하고 그 결과를 다시 입력으로 받아 다음 단계를 결정하는 단순한 반복 구조임.
- 상용 도구의 화려함에 압도되기보다 그 이면의 기술적 원리를 이해하면 누구나 자신만의 맞춤형 AI 코딩 에이전트를 구축할 수 있음.

# 상세 내용
- **Claude Code의 정체**: 앤스로픽이 최근 공개한 Claude Code는 터미널 기반의 에이전트로, 코드베이스를 분석하고 직접 수정하는 기능을 제공함. 하지만 기술적으로는 Claude 3.5 Sonnet 모델의 API 기능을 CLI 형태로 구현한 것임.
- **에이전트 루프(Agentic Loop)의 원리**:
  - 사용자의 질문을 입력받음.
  - 모델이 질문을 분석하고 필요한 도구(Tool)를 호출함.
  - 시스템이 해당 도구를 실제로 실행하고 결과를 모델에게 전달함.
  - 모델은 결과를 보고 작업을 끝낼지, 추가 도구를 사용할지 결정함.
- **필수적인 5가지 도구**:
  - list_files: 프로젝트의 파일 구조를 파악하기 위한 목록 나열.
  - read_file: 특정 파일의 내용을 읽어 컨텍스트 파악.
  - grep_search: 코드베이스 전체에서 특정 문자열이나 패턴 검색.
  - write_to_file: 분석 결과를 바탕으로 코드를 수정하거나 생성하여 저장.
  - run_terminal_command: 테스트 실행이나 빌드 등 임의의 명령어 수행.
- **시스템 프롬프트의 중요성**: 모델에게 유능한 소프트웨어 엔지니어라는 역할을 부여하고, 도구 사용 방식과 단계별 사고 과정을 지시하는 프롬프트가 에이전트 성능의 핵심임.
- **구현의 단순성**: 복잡한 프레임워크(LangChain 등) 없이도 Anthropic의 공식 SDK와 기본적인 파이썬 표준 라이브러리만으로 충분히 강력한 에이전트를 만들 수 있음.

# 인사이트 / 할 일
- **인사이트**: 최신 AI 도구들이 보여주는 '지능'은 사실 잘 설계된 도구 세트와 반복적인 루프의 결과물인 경우가 많음. 기술의 실체를 파악하면 막연한 두려움 대신 이를 활용할 아이디어가 생김.
- **인사이트**: 상용 도구는 범용성을 지향하지만, 직접 만든 도구는 내 특정 프로젝트 구조나 워크플로우에 완벽히 맞춤화할 수 있다는 장점이 있음.
- **할 일**: 글에서 제시된 200줄의 로직을 바탕으로, 내가 자주 반복하는 작업(예: 특정 라이브러리 업데이트, 보일러플레이트 생성)을 자동화하는 나만의 CLI 에이전트 제작해보기.
- **할 일**: Anthropic API의 'Tool Use' 문서를 숙지하여, 모델이 도구 실행 결과를 어떻게 처리하는지 기술적 세부 사항 확인하기.
