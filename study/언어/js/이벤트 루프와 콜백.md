
---
### 개요

- JS는 **싱글 스레드**라서 오래 걸리는 작업이 있으면 메인 스레드가 막힌다.
- 그래서 **브라우저(Web API)나 Node(libuv)**가 비동기 작업을 백그라운드에서 처리하고, 끝나면 콜백으로 결과를 돌려준다.

---

## 핵심 개념 정리

### 1) 비동기 작업

- 네트워크 요청, 파일 I/O처럼 시간이 오래 걸리는 작업을 **백그라운드에서 처리**하는 방식.

### 2) JS와 실행 환경

- JS 자체는 단일 스레드일 뿐이고, 비동기는 **브라우저/Node가 제공하는 엔진(Web API/libuv)**이 처리한다.

### 3) 콜백 함수

- 비동기 작업이 끝난 뒤 **메인 스레드에서 실행할 함수**를 전달하는 방식.

### 4) 이벤트 루프 동작 흐름

1. **Call Stack**: JS 코드 실행
2. 비동기 함수 호출 → **백그라운드로 위임**
3. 완료된 콜백 → **Event Queue**에 저장
4. **Call Stack이 비면** Event Loop가 Queue에서 콜백을 가져와 실행

### 5) 모던 비동기 방식

- **Promise / async-await**은 콜백 패턴을 개선한 문법적 발전형.

## 면접 답변 예시(짧고 깔끔하게)

“JS는 싱글 스레드라서 I/O 같은 긴 작업은 브라우저나 Node가 백그라운드에서 처리합니다. 작업이 끝나면 콜백이 이벤트 큐에 들어가고, 이벤트 루프가 콜스택이 비는 시점에 그 콜백을 실행합니다. 실무에서는 이를 Promise와 async/await으로 더 읽기 쉽게 다룹니다.”
